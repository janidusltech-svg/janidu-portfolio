Question 1 - Reading Comprehension

12 marks

Passage 1: The Importance of Clear Software Documentation

Software documentation is written information that explains how a software product should be installed, used, maintained or improved. It may include user manuals, installation guides, frequently asked questions, release notes and technical reference material. Good documentation is important because software users often need support after the product has been delivered.

Clear documentation reduces confusion and saves time. A user who can follow a step-by-step guide does not need to contact technical support for every small problem. In workplaces, this can reduce delays and improve productivity. For example, if employees receive clear instructions on how to use a new payroll system, they can complete routine tasks more confidently.

However, documentation is useful only if it is written for the correct audience. A guide for ordinary users should avoid unnecessary technical terms, while a guide for developers may include code examples and system requirements. Writers should therefore analyse the reader’s background knowledge before deciding how much detail to include.

Another key feature of effective documentation is accuracy. Screenshots, menu names and instructions must match the current version of the software. If the software is updated but the documentation is not revised, users may follow outdated instructions and make errors. Therefore, documentation should be reviewed whenever major changes are made to the product.

In short, software documentation is not an extra feature. It is a necessary part of software quality because it helps users understand the product, reduces support requests and improves the overall user experience.

Question 2 - Reading Comprehension

13 marks

Passage 2: Time Management among University Students

A. University students often face several academic responsibilities at the same time. They may have to attend lectures, complete assignments, prepare for tests and participate in group projects. Without good time management, these tasks can quickly become stressful.

B. One common problem is procrastination. Students sometimes delay difficult tasks because they feel uncertain or tired. Although postponing work may give temporary comfort, it usually creates greater pressure later. When deadlines are close, students may rush their work and produce lower-quality answers.

C. Planning can reduce this problem. A weekly study schedule helps students see how much time is available and which tasks are urgent. Large assignments should be divided into smaller stages, such as reading, outlining, drafting and editing. This makes the task easier to begin and easier to complete.

D. Time management also requires realistic priorities. Some students spend too much time on low-value activities, such as decorating slides, while ignoring the content that carries most marks. A useful strategy is to identify the most important tasks first and complete them before less important work.

E. Finally, rest should not be ignored. Students who study continuously without breaks may become tired and lose concentration. Short breaks, enough sleep and regular revision can help students work more effectively. Good time management is therefore not about studying every minute; it is about using time wisely.

Passage 3: Encouraging Responsible Use of Artificial Intelligence

Artificial intelligence tools are now used by many students for studying, writing and research. These tools can explain difficult ideas, suggest outlines and help users improve grammar. When used responsibly, they can support learning and save time.

However, AI tools also create risks. Students may copy AI-generated answers without understanding the content. This can lead to academic dishonesty and poor learning. Some AI responses may also contain inaccurate information, so students should check important facts using reliable sources.

Universities can encourage responsible AI use by giving clear guidelines. These guidelines should explain when AI assistance is allowed, how it should be acknowledged and which uses are not acceptable. Teachers can also design assessments that require personal reflection, class discussion or application of knowledge, making it harder for students to depend completely on AI.

Responsible AI use depends on honesty, critical thinking and proper guidance. AI should be treated as a learning support tool, not as a replacement for students’ own effort and understanding.
