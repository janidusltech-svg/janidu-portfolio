B1. RAM model and Big O (5 marks)
int count = 0;
for (int i = 1; i <= n; i++) {
    count = count + 1;
}
for (int j = 1; j <= 10; j++) {
    System.out.println(j);
}
Model answer: The first loop runs n times. Inside the loop, count = count + 1 is a constant-time operation. The second loop runs exactly 10 times, which is constant. Therefore, T(n) = c1n + c2(10) + c3. When using Big O, constants and lower-order terms are ignored. Final complexity = O(n). Marking guide: 2 marks for O(n). 1 mark for identifying first loop runs n times. 1 mark for identifying second loop is constant 10. 1 mark for explaining constants are ignored.
B2. Stack frame tracing (6 marks)

Initial: top = -1, stack size = 4. Operations:

push(5);
push(9);
peek();
push(12);
pop();
push(20);
Model answer: push(5) -> [5], top = 0 push(9) -> [5, 9], top = 1 peek() returns 9 but does not remove it -> [5, 9], top = 1 push(12) -> [5, 9, 12], top = 2 pop() removes 12 -> [5, 9], top = 1 push(20) -> [5, 9, 20], top = 2 Final stack = 5, 9, 20. Final top = 2. Marking guide: 2 marks for final stack. 1 mark for final top. 3 marks for correct step-by-step explanation.
B3. Circular queue tracing (6 marks)
maxSize = 6
front = 4, rear = 1, nItems = 4
Index:      0   1   2   3   4   5
Queue:      30  40  _   _   10  20
Logical queue = 10, 20, 30, 40

Operations:

remove();
insert(50);
insert(60);
Model answer: Initial logical queue = 10, 20, 30, 40. remove(): removes front item 10. New front = (4 + 1) % 6 = 5. nItems = 3. insert(50): new rear = (1 + 1) % 6 = 2. Insert 50 at index 2. nItems = 4. insert(60): new rear = (2 + 1) % 6 = 3. Insert 60 at index 3. nItems = 5. Final front = 5, rear = 3, nItems = 5. Final logical queue = 20, 30, 40, 50, 60. Marking guide: 1 mark each for front, rear, nItems. 2 marks for final logical queue. 1 mark for using circular formulas.

60, 40, 80, 30, 50, 70, 90

Section C — Essay Question 1: Data Structures (25 Marks)

C1(a). Stack application: reverse each word (8 marks)

Write a Java/pseudocode method to reverse each word in a string using only a stack. Example: "Data is fun" → "ataD si nuf"

Model answer: String reverseWords(String input) { String output = ""; StackX st = new StackX(input.length()); for (int i = 0; i < input.length(); i++) { char ch = input.charAt(i); if (ch != ' ') { st.push(ch); } else { while (!st.isEmpty()) { output = output + st.pop(); } output = output + " "; } } while (!st.isEmpty()) { output = output + st.pop(); } return output; } Explanation: Each word is pushed character by character. Because stack is LIFO, popping gives the characters in reverse order. When a space is found, the current word is popped and added to output. Marking guide: 2 marks for pushing non-space characters. 2 marks for popping at spaces. 2 marks for popping the last word after loop. 1 mark for using LIFO explanation. 1 mark for returning/outputting final string.
C1(b). Queue and circular queue code/theory (5 marks)
Model answer: A linear queue implemented using an array can waste free spaces at the front after several remove operations. Even if there are empty positions at the beginning, the rear may reach the last index and insertion may fail. A circular queue solves this by wrapping rear and front back to index 0 using modulo. rear = (rear + 1) % maxSize front = (front + 1) % maxSize Using nItems: boolean isEmpty() { return nItems == 0; } boolean isFull() { return nItems == maxSize; } Marking guide: 1 mark for explaining linear queue limitation. 1 mark for explaining circular wrap-around. 1 mark for rear formula. 1 mark for isEmpty. 1 mark for isFull.
C1(c). Linked list method writing (5 marks)

Write a method int countNodes() for a singly linked list.

Model answer: int countNodes() { int count = 0; Link current = first; while (current != null) { count++; current = current.next; } return count; } Explanation: Start from first. Visit each node until current becomes null. Increase count for every visited node. Marking guide: 1 mark for initializing count. 1 mark for starting current at first. 1 mark for while(current != null). 1 mark for count++ and current=current.next. 1 mark for returning count.
C1(d). BST construction, traversal, deletion (7 marks)

Insert into a BST: 45, 25, 65, 15, 35, 55, 75, 30, 40

Model answer: BST: 45 / \ 25 65 / \ / \ 15 35 55 75 / \ 30 40 Inorder = 15,25,30,35,40,45,55,65,75. Preorder = 45,25,15,35,30,40,65,55,75. Deleting 35: Node 35 has two children: 30 and 40. Replace 35 using its inorder successor. The inorder successor is the smallest value in the right subtree, which is 40. Then remove the old 40 node. Marking guide: 2 marks for inorder. 2 marks for preorder. 1 mark for correct tree drawing. 1 mark for identifying two-child deletion case. 1 mark for successor = 40.

Section C — Essay Question 2: Algorithms (25 Marks)

C2(a). Big O from running time equations (6 marks)
Model answer: T(n)=7n+4 -> O(n) T(n)=n²+100nlogn -> O(n²) T(n)=5logn+20 -> O(log n) T(n)=2ⁿ+n³ -> O(2ⁿ) Rule: In Big O notation, constants are ignored and the fastest-growing term is selected. Marking guide: 4 marks for correct answers. 1 mark for ignoring constants. 1 mark for selecting the highest growth term.
C2(b). Merge Sort recurrence and performance (5 marks)
Model answer: Merge Sort divides the array into two halves and then merges the sorted halves. Recurrence equation: T(n) = 2T(n/2) + O(n). The O(n) part is for merging. Using Master Theorem, this becomes O(n log n). Merge Sort has similar performance in best, average and worst cases because it always divides the array into halves and still performs the merge operation over all elements. Marking guide: 2 marks for recurrence. 1 mark for O(n log n). 1 mark for explaining division into halves. 1 mark for explaining merge work always happens.
C2(c). Quick Sort partition illustration (6 marks)
A = {9, 4, 7, 3, 8}
Pivot = last element
Model answer: A = [9,4,7,3,8], pivot = 8. Items less than pivot must go to the left side. 9 < 8? No. 4 < 8? Yes. Move/swap 4 to left -> [4,9,7,3,8] 7 < 8? Yes. Move/swap 7 to left -> [4,7,9,3,8] 3 < 8? Yes. Move/swap 3 to left -> [4,7,3,9,8] Final swap pivot 8 with first greater element 9 -> [4,7,3,8,9] Pivot final position = 4 using 1-based indexing. Important: Partition does not fully sort the left side. It only places smaller values left and bigger values right. Marking guide: 2 marks for final array. 1 mark for pivot position. 1 mark for pivot = 8. 1 mark for moving values smaller than pivot left. 1 mark for explaining partition is not full sorting.
C2(d). Heapify and BUILD_MAX_HEAP (5 marks)
Model answer: BUILD_MAX_HEAP starts from floor(n/2) because nodes after floor(n/2) are leaf nodes. Leaf nodes already satisfy the heap property, so heapify is needed only from the last non-leaf node up to the root. MAX_HEAPIFY compares a parent with its left and right children. The largest value among the parent and children is selected. If a child is larger than the parent, they are swapped and MAX_HEAPIFY is called again on the affected child position. BUILD_MAX_HEAP complexity = O(n). Marking guide: 2 marks for floor(n/2). 1 mark for O(n). 1 mark for explaining leaf nodes. 1 mark for explaining heapify comparison/swap.
